编译Protobuf(On Mac)

2015年11月14日

Protobuf推出3.0 beta版本了，相比2.0版提供了更多的语言支持，和更丰富的功能，2.0版只支持C++, JAVA, Python，3.0增加了对：Go, JavaNano, Ruby, and C#,等语言的支持，还会有更多的语言被支持。

想要使用最新版的protobuf，需要我们自行编译protoc，首先将最新代码下载到本地

然后根据官方文档编译就可以了，主要执行以下几步：

执行autogen.sh的时候，会自动下载：https://googlemock.googlecode.com/files/gmock-1.7.0.zip 这个文件，如果你不能翻墙，则autogen.sh命令将会由于超时而执行失败，权宜之计是自己下载gmock-1.7.0.zip这个文件将其丢到protobuf源代码目录下，然后编辑autogen.sh文件，找到curl $curlopts -O https://googlemock.googlecode.com/files/gmock-1.7.0.zip这一行，将其注释后再运行./autogen.sh即可。

执行autogen.sh过程可能会出现很多错误，这大多是缺少相关依赖程序造成的，依赖的程序主要由：autoconf, automake, libtool。安装过程中可能出现的错误信息有：

Failed to connect to googlemock.googlecode.com port 443: Operation timed out

是因为googlecode.com被墙了，需要翻墙！

./autogen.sh: line 43: autoreconf: command not found

是因为autoconf没有安装，执行：brew install autoconf进行安装

Can’t exec “aclocal”: No such file or directory at /usr/local/Cellar/autoconf/2.69/share/autoconf/Autom4te/FileUtils.pm line 326.

这是因为autoconf依赖的automake没有安装，执行：brew install automake

Can’t exec “glibtoolize”: No such file or directory at /usr/local/Cellar/autoconf/2.69/share/autoconf/Autom4te/FileUtils.pm line 345, line 6.
autoreconf: failed to run glibtoolize: No such file or directory
autoreconf: glibtoolize is needed because this package uses Libtool

这是因为autoconf依赖的libtool没有安装，执行：brew install libtool
